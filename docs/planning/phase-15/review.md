## Risks / Rollback
- Existing lint warnings remain in unrelated app screens. Mitigation: keep them out of Phase 15 scope; address in a separate cleanup phase.
- The approved-seed following lane still depends on the third-party following actor’s available metadata, so `seedCreatorId` persistence is limited by upstream output quality. Mitigation: preserve the lane and provenance model, then deepen per-seed attribution only if the actor exposes that metadata reliably.
- Rollback path: revert the Phase 15 commits and re-run `cd apps/web && npm run db:push` against the prior schema state before using the older discovery runtime.
