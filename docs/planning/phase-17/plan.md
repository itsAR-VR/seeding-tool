## RED TEAM Findings (Gaps / Weak Spots)

### Highest-risk failure modes

1. **Silent downstream data loss (CRITICAL)** — `deriveBrandICP()` at `icp.ts:114-130` does property-name lookups on the raw `brandProfile` JSON for `targetAudience`, `audience`, `niche`, `category`, `tone`, `brandVoice`. If the Business DNA schema uses different field names (e.g., `target_audience` instead of `targetAudience`), ICP derivation silently returns null for everything. → **Fix:** The Business DNA schema MUST include at minimum: `targetAudience`, `niche`, `category`, `tone`, `brandVoice` as top-level fields. Add explicit compatibility test in 17d.

2. **Request timeout blow-up** — Current `fetchBrandProfile` uses 8s timeout. Adding GPT analysis in the same synchronous POST request means total time = 8s (fetch) + 10-30s (model) ≈ 18-38s. The current UI progress animation cycles through 3 steps every 1.3s × 3 = 3.9s and will loop multiple times looking stale. → **Fix:** Plan must specify (a) model call timeout (recommend 20s), (b) total request timeout budget, and (c) whether the analysis is synchronous in the POST or kicked off as a background step with polling.

3. **Onboarding page concurrent edit collision** — `page.tsx` has 51 insertions / 91 deletions of uncommitted changes from Phase 16 work. Phase 17c will heavily rewrite this same file. → **Fix:** Phase 17c MUST re-read the file immediately before editing. The plan should note that 17c begins from the current working-tree state, not from the committed state.

### Missing or ambiguous requirements

4. **Model name verified** — User confirmed `gpt-5-mini`. All existing `gpt-4o-mini` references in the codebase will be updated to `gpt-5-mini` as part of this phase.

5. **API pattern decision unmade** — Plan says `openai.responses.parse(...)` but the codebase exclusively uses `openai.chat.completions.create()`. Both work with SDK v6.27+ but mixing patterns adds cognitive load. → **Fix:** Lock the decision: either adopt `responses.parse` for this call only (and document why) or stay consistent with `chat.completions.create` + `response_format: { type: "json_schema" }`.

6. **Storage decision deferred to 17a audit** — Subphase 17a work item 4 says "Decide whether this schema lives inside `BrandSettings.brandProfile`..." but this must be decided BEFORE implementation starts because downstream consumers cast `brandProfile` as `BrandProfileSnapshot`. → **Fix:** Lock decision now: extend the existing `brandProfile` JSON blob with Business DNA fields merged at the top level (so `deriveBrandICP` field-name lookups work). The raw `BrandProfileSnapshot` fields stay in place; new synthesis fields are added alongside them.

7. **New analysis helper file path unspecified** — Subphase 17b says "Add a dedicated server-side analysis helper" but doesn't say where. → **Fix:** Place at `apps/web/lib/brands/synthesis.ts` (alongside `profile.ts` and `icp.ts` in the brands lib).

8. **Image extraction gap — RESOLVED** — Plan mentions "candidate media/image URLs" but `fetchBrandProfile` only extracts `ogImage` and `twitterImage`. → **Fix:** Add hero `<img>` extraction from `<main>`/first `<section>` in 17a/17b. Filter rules: exclude SVGs, tracking pixels, images with width or height < 100px. Include as `heroImageCandidates: string[]` in the profile.

### Performance / timeouts

9. **No model call timeout specified** — The plan says to call GPT but doesn't set a timeout. The fetch already has 8s. → **Fix:** Add explicit `AbortSignal.timeout(20_000)` or equivalent for the model call, and document the total worst-case budget: 8s fetch + 20s model = 28s.

10. **No streaming/progressive display** — 28s synchronous wait with a 3-step animation loop will feel broken. → **Fix:** Either (a) add more analysis steps to the animation to cover 30s, or (b) show real progress by streaming/polling, or (c) add a "this might take a moment" message after 5s.

### Security / permissions

If the key is missing (local dev, test env), the degraded path must handle this gracefully. → **Fix:** Check `process.env.OPENAI_API_KEY` before attempting synthesis; if missing, skip synthesis and persist raw profile only (log a warning).
