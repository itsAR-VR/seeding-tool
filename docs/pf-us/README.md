# P&F U.S. workspace artifacts

This subtree keeps P&F-specific working artifacts together instead of scattering them across root docs paths.

## Layout
- `context/` — truth-pass briefs, positioning notes, answer context
- `logs/` — execution logs or evidence snapshots worth keeping
- `memory/` — durable internal notes that are safe to keep in-repo

The goal is simple: context stays findable, and repo root stays less cursed.
